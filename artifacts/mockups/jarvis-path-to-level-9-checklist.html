Canonical Frame

Always-On Orchestrator, Not Assistant

JARVIS is a continuously running society of specialized agents working on behalf of the household. Direct interaction is only one operating mode. The real product is a governed intelligence environment that preserves continuity, reduces entropy against intention, and earns authority through evidence instead of assuming it.

Primary Constraint Event truth, safe persistence, and governed authority must become real before broader autonomy can be trusted.
Core Mechanism The promotion engine is the bridge from sandbox work to earned live authority.
Final Test Level 9 exists only when JARVIS is household-operable, memory-compounding, inspectable, and formative across years.
Milestone Ladder
Level 3 Household operating system with proactive daily orchestration.
Level 4 Governed intelligence system with trust zones, stages, and auditability.
Level 5-6 Ambient delivery plus durable institutional memory and continuity.
Level 7-9 Formation, bounded autonomy, and family civilization infrastructure.
Phase A

Finish The Level 3 Foundation

Get the live product coherent enough that JARVIS behaves like one system instead of a collection of partly-wired surfaces.

Now Gate To Level 4

Checklist

  • Shared-state coherence across web, phone, and runtimeRemove split-brain behavior so the household sees one truth surface and one orchestration state.
  • Finish live wiring for house, calendar, perception, and workshopReplace profile-backed seams with real household integrations where the tracker still shows simulation or missing credentials.
  • Apply the always-on deployment footprint to real hardwareMove from documented launchd and infra posture to actual always-on household runtime behavior.

Exit Gate

  • Real daily operating trustThe family can run a real day through JARVIS without hidden simulation boundaries breaking the experience.
  • Live wiring truthBlockers around Home Assistant, perception hardware, reminders/calendar, and workshop telemetry are reduced to edge cases, not core posture.
Phase B

Build The Always-On Orchestrator Core

Make the agent society load-bearing instead of metaphorical.

Now Load-Bearing

Checklist

  • Durable agent lifecycle kernelWake, run, pause, resume, interrupt, escalate, retire, and survive restart without losing run state.
  • Agent registry as source of truthEvery agent needs identity, mission, cadence, trust zone, stage, stewardship lane, and escalation target.
  • Event bus and scheduler fabricAgents must wake on cadence, signals, thresholds, state changes, handoffs, and direct human interruption.
  • Foreground/background attention routerDirect conversation becomes one focus mode inside continuous background work instead of replacing it.
  • Operator observability planeMake active, blocked, stale, overloaded, and attention-seeking agents visible without opening code.

Exit Gate

  • Society becomes realJARVIS can accurately say which agents are awake, blocked, idle, stale, or escalating and why.
  • No session illusionBackground work persists and foreground interaction no longer resets the system’s identity.
Phase C

Make Event Truth And Persistence Load-Bearing

Turn event history into the substrate, not a debugging sidecar.

Now Substrate

Checklist

  • Append-only typed event historyState change, overrides, promotions, failures, and agent actions all become versioned event records.
  • Projected operational stateRuntime status, queues, supervision, and later memory views should derive from event truth instead of ad hoc mutable blobs.
  • Concurrency-safe persistenceBackground agents must not silently corrupt queues, run records, or promotion evidence under concurrent writes.
  • Replay, explanation, rollback, and auditabilityThe system must explain why it acted, reconstruct history, and unwind bounded actions with traceable provenance.

Exit Gate

  • Trustworthy recoveryRestart and replay can reconstruct the current picture without inventing missing state.
  • Evidence foundationPromotion, demotion, suspension, and learning all have a durable substrate.
Phase D

Make Governance Real In Code

Trust zones and authority stages stop being doctrine-only and become runtime law.

Next Level 4

Checklist

  • Trust-zone control plane enforcementEvery consequential action path must resolve against zone, stage, arena, and boundary rules in code.
  • Authority stages and escalation boundariesObserve, draft, stage, sandbox live, mature live, and suspended must materially change what the system can do.
  • Rollback, demotion, and suspension pathsUnsafe or drifting behavior must reduce authority instead of merely being noted in logs.
  • Action paths cite authority and event trailEvery meaningful mutation should be explainable in terms of scope, stage, basis, and reviewed evidence.

Exit Gate

  • Governed intelligence systemJARVIS can take initiative, but only inside explicit and inspectable authority boundaries.
  • No hand-wavy autonomyAutonomous language cannot outrun what the runtime actually enforces.
Phase E

Build The Real Agent Society

Give the agents durable collaboration primitives instead of letting them behave like isolated helpers.

Next Coordination

Checklist

  • Durable inbox, outbox, worklog, blocked-state, and hypothesis stateAgents need their own operating memory instead of only shared aggregate surfaces.
  • Cross-agent delegation and review requestsAgents should be able to hand work, request evidence, merge duplicate efforts, and escalate cleanly.
  • Visible stewardship lanesOrganize agents into human-readable ownership lanes with health, cadence, and burden-relief framing.
  • Resource arbitrationAlways-on work must compete sanely for compute, tokens, trust headroom, and human attention.

Exit Gate

  • Coordinated staff behaviorThe system behaves like a staff with lanes, handoffs, and reviews instead of a pile of isolated prompts.
  • Operator trustThe user can tell what the society is doing without reading raw implementation state.
Phase F

Build The Promotion Engine

Convert track record into authority and make earned autonomy a real mechanism. Current slice landed: evaluation, recommendation, governed apply, and batch apply.

Now Earned Authority

Checklist

  • Run records, promotion claims, and verdictsCreate the canonical objects that allow the system to argue for more authority with evidence instead of aspiration.
  • Sandbox evidence before higher authorityBroader power should be unlocked only after bounded runs prove repeatable, reviewable success.
  • Human consent only at the mature-live gateLet useful motion happen earlier, but require explicit consent before deeper lasting authority.
  • Promotion scoring feeds from failures and overrides tooRollbacks, reversals, rejected reviews, and suspension signals should matter just as much as wins.

Exit Gate

  • Legitimate autonomyThe system earns the right to do more instead of claiming it because a policy file said so.
  • Promotion queueOperator surfaces can show what should be promoted, held, demoted, or suspended right now.
Phase G

Make JARVIS Household-Operable

Break the key-person dependency so the system can become a household civilization layer instead of a one-builder console.

Next Legitimacy

Checklist

  • Role-aware surfaces for the familyDifferent people should get different levels of complexity, authority, and visibility without needing builder knowledge.
  • Trusted bounded controls for non-buildersNormal review, correction, and steering tasks should be possible without touching raw admin concepts.
  • Rebekah-usable command, review, and correction flowsThe household system must work when the builder is absent, overloaded, or not the person handling the situation.
  • Reduced operator fragilityRuntime management, approvals, and trust-stage visibility must survive one person being unavailable.

Exit Gate

  • Household-operableJARVIS no longer depends on one person to interpret and drive every important surface.
  • Category credibilityOnly now can the system claim to be a household intelligence layer instead of a sophisticated personal rig.
Phase H

Build Level 5 And Level 6 Behaviors

Make JARVIS ambient, disciplined, and truly continuity-bearing.

Later Ambient + Memory

Checklist

  • Ambient but disciplined deliveryRoute interventions across phone, watch, home, and room surfaces without becoming another noisy notification system.
  • Presence-aware interruption logicUrgency, quiet hours, focus mode, and household context must decide when agents can surface themselves.
  • Institutional memory beyond stored factsCommitments, rituals, patterns, prior resolutions, and family-specific history must become usable continuity.
  • Long-running agents use memory, not session-local recallThe society should get smarter because it remembers, not because the current prompt got longer.

Exit Gate

  • Ambient household intelligenceJARVIS helps before being asked while still preserving peace, trust, and quiet.
  • Continuity enginePast decisions begin to materially improve present ones in a traceable way.
Phase I

Build Level 7 And Level 8 Behaviors

Expand from operational support into formation and bounded real work.

Later Formation + Bounded Autonomy

Checklist

  • Formation loops as first-class systemsTutoring, parenting, health, faith, stewardship, creativity, and growth must become structural, not peripheral.
  • Bounded autonomous operatorsLet JARVIS do selected real work inside governed arenas where the risk class and rollback posture are explicit.
  • Sandboxable, promotable execution pathsPublishing, drafting, monitoring, and reversible execution should be measurable, reviewable, and eligible for earned promotion.

Exit Gate

  • Formation platformJARVIS is helping the family become, not only operate.
  • Bounded autonomous operatorThe system can execute real work without dropping governance or reversibility.
Phase J

Reach Level 9

Treat JARVIS as family civilization infrastructure rather than a feature product.

Later Family Civilization Layer

Checklist

  • Intergenerational memory and operational valuesFamily identity, wisdom, patterns, and values become durable operating logic instead of hopeful sentiment.
  • Continuity resilient across yearsThe system compounds memory, legitimacy, and stewardship without depending on one season, one repo shape, or one operator.
  • Inspectability and override remain intactEven at civilization scale, the household can still inspect, refuse, redirect, and bound what JARVIS is doing.

Exit Gate

  • Family civilization layerJARVIS strengthens identity, judgment, continuity, and stewardship over long horizons.
  • Deep promise keptyou're not carrying this alone becomes operational truth, not brand language.
Decision Complete

What Matters Most First

If this checklist has to be compressed into one ruthless order, the sequence is: evented runtime, safe persistence, governance enforcement, promotion engine, household-operable surfaces, then ambient memory and formative autonomy.

  • Do not broaden autonomy before event truth, persistence safety, and rollback posture are real.
  • Do not claim household intelligence if the household cannot operate it directly.
  • Do not chase Level 9 language while Level 4 authority and Level 6 continuity are still thin.
