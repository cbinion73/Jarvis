Mobile Command Layer

JARVIS on iPhone is a silent-first operating surface.

The phone should not behave like a shrunken desktop or a voice-led chat app. It should help with fast review, one-tap action, continuity, family logistics, movement-aware triage, and real-world decision making when speaking is inconvenient or impossible.

Primary Job Show what changed, what matters now, and what is blocked on the user.
Primary Feel Calm, trustworthy, glanceable, and ready for one-handed silent action.
Primary Trap To Avoid Recreating fourteen equal tabs and calling it mobile strategy.

Resume is intentionally not its own tab. It should appear as continuity cards across Today, Decisions, Family, and Navigate so users can recover context without re-navigating the whole app.

Silent-First Rules

These interaction rules keep the mobile product grounded in real-life, no-speaking-required use.

Glanceable Severity, deadline, and next action must be visible within a two-second scan.
One-Tap First Approve, defer, pin, acknowledge, and resume should usually beat typing.
Resume Native Returning users should see “resume” paths instead of being forced to find the right tab again.
Voice Optional Voice entry is a useful accelerant for capture and in-motion use, never the only path.

Mobile vs Desktop

The phone should be sharper and smaller in scope, not equally broad with less room.

Desktop Wins
  • Deep agent supervision
  • Longer review sessions
  • Broader workspace inspection
  • Systems and doctrine depth
Mobile Wins
  • Urgency triage
  • One-tap approvals
  • Commute and movement steering
  • Absence recovery and continuity
If mobile ends up exposing every domain as a peer in primary navigation, it is drifting back toward a mini desktop. The command layer should stay opinionated.
